\begin{abstract}
We present a reproducible benchmark for evaluating open instruction-tuned large language models on Greek NLP. The benchmark covers grammatical error correction, intent classification, legal document classification, machine translation, named entity recognition, part-of-speech tagging, and summarization. We evaluate five open models under a deterministic capped-test protocol that preserves ordering and exact reproducibility while reducing the runtime burden of very large test sets. Across nine benchmark segments, \texttt{gemma2:9b} ranks first overall on normalized quality, while \texttt{aya-expanse:8b} is strongest on legal classification and English machine translation, and \texttt{llama3.1:8b-instruct} performs best on summarization. The paper is paired with repository-native tables, vector figures, and combined prediction exports so that benchmark execution and manuscript preparation remain tightly coupled.
\end{abstract}

\section{Introduction}
Broad evaluation has become central to modern language model research because single-task scores rarely capture deployment-relevant trade-offs in robustness, latency, and transfer \cite{bommasani2021foundation,liang2023helm,gao2023lmeval}. Yet, for Greek NLP, evaluation resources still remain fragmented across task-specific datasets, metrics, and execution pipelines. This makes it difficult to compare open instruction-tuned LLMs under a shared protocol, especially when the goal is not a single downstream task but a broader view across heterogeneous settings.

Instruction tuning has made general-purpose language models more usable across many tasks without task-specific retraining \cite{wei2022finetuned,chung2024scaling}. However, evaluating such models on Greek remains cumbersome in practice. This paper packages our evaluation workflow into a reproducible benchmark suitable for SETN 2026. The benchmark aggregates seven task families into nine reporting segments: grammatical error correction (GEC), intent classification, legal classification, machine translation into three target languages, named entity recognition (NER), part-of-speech (POS) tagging, and summarization. The runner produces per-task summaries, per-instance predictions, combined CSV outputs, HTML visualizations, and publication-facing figures.

Our contribution is practical rather than algorithmic. First, we provide a unified multi-task evaluation setup for Greek NLP with open local models. Second, we adopt a deterministic capped-test protocol that remains reproducible without relying on random sampling. Third, we expose benchmark outputs in a form that flows directly into a conference paper.

\section{Benchmark Setup}
\subsection{Tasks and Models}
The benchmark covers seven task families:
\begin{itemize}[leftmargin=1.2em]
\item GEC
\item intent classification
\item legal classification
\item machine translation
\item NER
\item POS tagging
\item summarization
\end{itemize}

We evaluate five open instruction-tuned models served locally through Ollama:
\texttt{qwen2.5:7b-instruct},
\texttt{llama3.1:8b-instruct},
\texttt{aya-expanse:8b},
\texttt{gemma2:9b}, and
\texttt{falcon3:7b-instruct}.

\subsection{Deterministic Capped-Test Protocol}
The full test sets are very uneven in size, with legal classification and machine translation being especially large. To keep evaluation tractable while preserving reproducibility, we use a deterministic capped-test profile that always takes the first instances in dataset order instead of random samples. The resulting capped evaluation contains:
\begin{itemize}[leftmargin=1.2em]
\item GEC: 175 examples
\item intent classification: 436 examples
\item legal classification: 500 examples
\item machine translation: 300 English pairs, 4 Persian pairs, and 9 Japanese pairs
\item NER: 500 examples
\item POS: 456 examples
\item summarization: 300 examples
\end{itemize}

This design keeps the smaller tasks intact while reducing the runtime burden of the largest tasks. It also supports exact reruns because the selected subset is deterministic.

\subsection{Metrics}
We report task-appropriate primary metrics. For classification-style tasks we use accuracy and macro-F1, with macro-F1 treated as the primary comparison metric. For GEC we report exact match and edit-distance-derived measures against the reference and source text. For machine translation and summarization we currently rely on word error rate (WER) and character error rate (CER), where lower values are better. Latency is reported as average per-instance generation time in seconds.

To compare models across heterogeneous metrics, we additionally compute a direction-aware normalized quality score per benchmark segment. Within each segment, the best model receives the highest normalized value and the worst model the lowest. These segment-normalized scores are then averaged to obtain an overall ranking.

\section{Implementation Notes}
The benchmark pipeline was adjusted in several ways to make end-to-end runs stable and paper-ready. We added combined prediction exports so that the full suite produces unified CSV files across tasks. We also changed legal classification from opaque subject identifiers to coarse-grained human-readable volume labels, which avoids the degenerate all-zero behavior observed with raw numeric class IDs. For POS tagging, we added a fallback parser that tolerates \texttt{\_} head values in CoNLL-U files. At the backend level, we introduced retry and backoff logic around transient Ollama failures. Finally, we generated publication-oriented reports, tables, and vector graphics directly from benchmark outputs.

Three patterns stand out. First, \texttt{gemma2:9b} is the strongest overall model in this run. It ranks first on the normalized aggregate and wins four benchmark segments, namely GEC, intent classification, NER, and POS. Second, \texttt{aya-expanse:8b} is especially competitive on cross-lingual tasks, winning legal classification and English machine translation, and tying for the best Japanese machine translation WER. Third, \texttt{llama3.1:8b-instruct} performs best on summarization, although its advantage comes at a higher latency than the fastest summarization model.

Legal classification remains the most difficult benchmark component. Even after replacing opaque identifiers with coarse-grained volume labels, the best macro-F1 is only 0.011. This suggests that the task is still demanding under zero-shot prompting and may benefit from better label descriptions, exemplars, or task-specific fine-tuning. By contrast, intent classification is comparatively mature in this setup, with \texttt{gemma2:9b} reaching 0.737 macro-F1 on 436 capped test examples.

Machine translation is mixed. \texttt{aya-expanse:8b} is clearly strongest for English translation, but the Persian and Japanese segments are extremely small in this deterministic capped profile. As a result, conclusions for those two languages should be treated as suggestive rather than definitive.

\section{Discussion}
The benchmark is useful because it combines multiple Greek NLP settings in a single reproducible workflow, but the current evaluation still has limitations. The capped protocol is intentionally pragmatic rather than exhaustive. It is well suited for iterative comparison and paper-scale reporting, but it is not a substitute for full test-set evaluation when final archival numbers are required. Some metric choices are also provisional. In particular, summarization and machine translation would benefit from stronger semantic metrics and human assessment, while legal classification would benefit from richer label semantics.

Despite these caveats, the benchmark already exposes meaningful structure in model behavior. Some models offer strong general performance across multiple tasks, while others are better understood as specialists for translation or summarization. This kind of cross-task perspective is hard to obtain from isolated task runs and is exactly where a unified benchmark becomes valuable.

\section{Reproducibility and Artifact Structure}
The repository includes the benchmark runner, combined CSV outputs, HTML task visualizations, report generation scripts, and publication-ready assets. The capped run used in this draft is stored under the repository's completed-run hierarchy, and the article-facing tables and vector figures are generated into a dedicated \texttt{article\_assets} subdirectory. This structure allows the manuscript to be regenerated directly from benchmark outputs without manual copy-paste into the paper.

\section{Conclusion}
We presented a reproducible Greek NLP benchmark draft for open instruction-tuned LLMs and used it to compare five models across seven task families and nine benchmark segments. Under a deterministic capped-test protocol, \texttt{gemma2:9b} ranks first overall, while \texttt{aya-expanse:8b} and \texttt{llama3.1:8b-instruct} emerge as strong specialists on legal classification and summarization respectively. The current pipeline is already useful for comparative evaluation, and it also provides a clean path from benchmark execution to publication assets.
