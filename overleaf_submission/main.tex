\begin{abstract}
We present a reproducible benchmark for evaluating open instruction-tuned large language models on Greek NLP. The benchmark covers grammatical error correction, intent classification, legal document classification, machine translation, named entity recognition, part-of-speech tagging, and summarization. We evaluate five open models under a deterministic capped-test protocol that preserves ordering and exact reproducibility while reducing the runtime burden of very large test sets. Across nine benchmark segments, \texttt{gemma2:9b} ranks first overall on normalized quality, while \texttt{aya-expanse:8b} is strongest on legal classification and machine translation, and \texttt{llama3.1:8b-instruct} performs best on summarization. The paper is paired with repository-native tables, vector figures, appendix-level task analyses, and combined prediction exports so that benchmark execution and manuscript preparation remain tightly coupled.
\end{abstract}

\section{Introduction}
Broad evaluation has become central to modern language model research because single-task scores rarely capture deployment-relevant trade-offs in robustness, latency, and transfer \cite{bommasani2021foundation,liang2023helm,gao2023lmeval}. Yet, for Greek NLP, evaluation resources still remain fragmented across task-specific datasets, metrics, and execution pipelines. This makes it difficult to compare open instruction-tuned LLMs under a shared protocol, especially when the goal is not a single downstream task but a broader view across heterogeneous settings.

Instruction tuning has made general-purpose language models more usable across many tasks without task-specific retraining \cite{wei2022finetuned,chung2024scaling}. However, evaluating such models on Greek remains cumbersome in practice. This paper packages our evaluation workflow into a reproducible benchmark suitable for SETN 2026. The benchmark aggregates seven task families into nine reporting segments: grammatical error correction (GEC), intent classification, legal classification, machine translation into three target languages, named entity recognition (NER), part-of-speech (POS) tagging, and summarization. The runner produces per-task summaries, per-instance predictions, combined CSV outputs, HTML visualizations, and publication-facing figures.

Our contribution is practical rather than algorithmic. First, we provide a unified multi-task evaluation setup for Greek NLP with open local models. Second, we adopt a deterministic capped-test protocol that remains reproducible without relying on random sampling. Third, we expose benchmark outputs in a form that flows directly into a conference paper, including automatically generated appendix materials for task-level analysis.

\section{Benchmark Setup}
\subsection{Tasks and Models}
The benchmark covers seven task families:
\begin{itemize}[leftmargin=1.2em]
\item GEC
\item intent classification
\item legal classification
\item machine translation
\item NER
\item POS tagging
\item summarization
\end{itemize}

We evaluate five open instruction-tuned models served locally through Ollama:
\texttt{qwen2.5:7b-instruct},
\texttt{llama3.1:8b-instruct},
\texttt{aya-expanse:8b},
\texttt{gemma2:9b}, and
\texttt{falcon3:7b-instruct}.

\subsection{Deterministic Capped-Test Protocol}
The full test sets are very uneven in size, with legal classification and machine translation being especially large. To keep evaluation tractable while preserving reproducibility, we use a deterministic capped-test profile that always takes the first instances in dataset order instead of random samples. The resulting capped evaluation contains:
\begin{itemize}[leftmargin=1.2em]
\item GEC: 175 examples
\item intent classification: 436 examples
\item legal classification: 500 examples
\item machine translation: 500 English pairs, 500 Persian pairs, and 500 Japanese pairs
\item NER: 500 examples
\item POS: 456 examples
\item summarization: 300 examples
\end{itemize}

This design keeps the smaller tasks intact while reducing the runtime burden of the largest tasks. It also supports exact reruns because the selected subset is deterministic. For machine translation, the benchmark now uses a shared data-wrapper path rather than a separate task-local split implementation, which eliminates an earlier source of inconsistency in language-specific counts.

\subsection{Metrics}
We report task-appropriate primary metrics. For classification-style tasks we use accuracy and macro-F1, with macro-F1 treated as the primary comparison metric. For GEC we report exact match and edit-distance-derived measures against the reference and source text. For machine translation we use chrF as the primary reporting metric and retain WER/CER as supplementary string-distance diagnostics. For summarization we use BERTScore-F1 as the primary semantic comparison metric and report WER/CER and ROUGE as supplementary overlap-based diagnostics. We also compute GLEU for GEC and entity-level F1 for NER. Latency is reported as average per-instance generation time in seconds.

To compare models across heterogeneous metrics, we additionally compute a direction-aware normalized quality score per benchmark segment. Within each segment, the best model receives the highest normalized value and the worst model the lowest. These segment-normalized scores are then averaged to obtain an overall ranking. The main paper focuses on the primary metrics used for ranking, while Appendix~\ref{sec:appendix-per-task-analysis} reports the richer per-task metric panels.

\section{Implementation Notes}
The benchmark pipeline was adjusted in several ways to make end-to-end runs stable and paper-ready. We added combined prediction exports so that the full suite produces unified CSV files across tasks. We also changed legal classification from opaque subject identifiers to coarse-grained human-readable volume labels, which avoids the degenerate all-zero behavior observed with raw numeric class IDs. For POS tagging, we added a fallback parser that tolerates \texttt{\_} head values in CoNLL-U files. At the backend level, we introduced retry and backoff logic around transient Ollama failures. Finally, we generated publication-oriented reports, tables, vector graphics, and appendix-ready per-task analyses directly from benchmark outputs.

Four patterns stand out. First, \texttt{gemma2:9b} remains the strongest overall model in this run, ranking first on the normalized aggregate with an average quality score of 71.6 and winning GEC, intent classification, NER, POS, and two of the three machine translation segments under chrF. Second, \texttt{llama3.1:8b-instruct} rises to second overall once summarization is ranked semantically with BERTScore-F1 rather than by WER, which highlights the sensitivity of aggregate rankings to metric choice. Third, \texttt{aya-expanse:8b} remains especially competitive on cross-lingual tasks, winning legal classification, the English translation segment by WER's closest competitor metric chrF only narrowly losing to \texttt{gemma2:9b}, and taking the top summarization BERTScore-F1. Fourth, the weakest aggregate model is not uniformly weak: for example, \texttt{falcon3:7b-instruct} is often among the fastest models even when it lags in quality, which reinforces the value of reporting latency alongside task scores.

Legal classification remains the most difficult benchmark component. Even after replacing opaque identifiers with coarse-grained volume labels, the best macro-F1 is only 0.011. This suggests that the task is still demanding under zero-shot prompting and may benefit from better label descriptions, exemplars, or task-specific fine-tuning. By contrast, intent classification is comparatively mature in this setup, with \texttt{gemma2:9b} reaching 0.737 macro-F1 on 436 capped test examples. The appendix confirms that the legal task is difficult under both accuracy and macro-F1, not only under the headline metric.

Machine translation is mixed but now much more interpretable than in our earlier smaller-language slices. The capped profile evaluates 500 examples per target language, which is enough to separate models consistently while keeping runtime manageable. Using chrF as the primary metric changes the winners relative to WER: \texttt{gemma2:9b} leads English and Japanese translation, while \texttt{qwen2.5:7b-instruct} jumps to first on Persian because its chrF substantially exceeds the rest of the field despite weaker WER. This is precisely the kind of metric dependence that argues for reporting both primary and supplementary MT metrics in the appendix rather than relying on a single surface-form score.

The appendix also surfaces secondary structure in tasks where the main ranking alone would be too coarse. For NER, \texttt{llama3.1:8b-instruct} has the highest accuracy while \texttt{gemma2:9b} leads on macro-F1, suggesting differences in minority-entity handling. For summarization, the shift from WER to BERTScore-F1 changes the segment winner from \texttt{llama3.1:8b-instruct} to \texttt{aya-expanse:8b}, but only by a very small margin (0.519 vs.\ 0.517), which suggests that the top two models are effectively near-tied semantically even though the overlap-based and distance-based metrics tell somewhat different stories.

\section{Discussion}
The benchmark is useful because it combines multiple Greek NLP settings in a single reproducible workflow, but the current evaluation still has limitations. The capped protocol is intentionally pragmatic rather than exhaustive. It is well suited for iterative comparison and paper-scale reporting, but it is not a substitute for full test-set evaluation when final archival numbers are required. Some metric choices are also provisional. In particular, summarization and machine translation would benefit from stronger semantic metrics and human assessment, while legal classification would benefit from richer label semantics. Even so, the appendix-level supplemental metrics already improve interpretability relative to a WER-only or macro-F1-only summary.

Despite these caveats, the benchmark already exposes meaningful structure in model behavior. Some models offer strong general performance across multiple tasks, while others are better understood as specialists for translation or summarization. This kind of cross-task perspective is hard to obtain from isolated task runs and is exactly where a unified benchmark becomes valuable.

\section{Reproducibility and Artifact Structure}
The repository includes the benchmark runner, combined CSV outputs, HTML task visualizations, report generation scripts, and publication-ready assets. The capped run used in this draft is stored under the repository's completed-run hierarchy, and the article-facing tables and vector figures are generated into a dedicated \texttt{article\_assets} subdirectory. This structure allows the manuscript to be regenerated directly from benchmark outputs without manual copy-paste into the paper.

\section{Conclusion}
We presented a reproducible Greek NLP benchmark draft for open instruction-tuned LLMs and used it to compare five models across seven task families and nine benchmark segments. Under a deterministic capped-test protocol, \texttt{gemma2:9b} ranks first overall, while \texttt{aya-expanse:8b} and \texttt{llama3.1:8b-instruct} emerge as strong specialists on legal classification and summarization respectively. The current pipeline is already useful for comparative evaluation, and it also provides a clean path from benchmark execution to publication assets.

\section{Prompt Templates}
This appendix lists the system prompts and user-side task templates used in the benchmark. Concrete examples are instantiated by replacing angle-bracket placeholders with task-specific content at runtime.

\subsection{GEC}
\paragraph{System prompt}
\begin{verbatim}
You are a careful grammatical error correction system for Modern Greek.
Correct the user's sentence while preserving its meaning.
Return only the corrected Greek sentence and nothing else.
\end{verbatim}
\paragraph{Prompt template}
\begin{verbatim}
Diorthose to parakato elliniko keimeno grammatika kai orthografika.
Kratise to idio noima.

Keimeno: <ORIGINAL_TEXT>
Diorthomeno keimeno:
\end{verbatim}

\begin{verbatim}
